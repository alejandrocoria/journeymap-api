Requirements:

- Minecraft 1.5
- ModLoader or Forge

Changes in JourneyMap 2.1 for Minecraft 1.5:
- New Feature (5ZA-ZYN-B26M): Displays Rei's Minimap or VoxelMap (formerly Zan Minimap) waypoints
- Improved Nether, End, and Cave mapping with contour shading
- Entities now remain visible while dragging the in-game map, releasing the drag no longer causes "jitter" 
- In-game map UI adjusted to make off-screen waypoint markers easier to see
- Map options in-game now shown over the map
- Map options now saved in journeyMap.properties file and synchronized between web UI and in-game UI
- Added hover text to Web UI menu items
- Tweaked Web UI appearance of World Info area
- Improved rendering of End worlds
- Radar range (for mobs, players, etc) has been extended to be 1/2 of the mapping radius (horizontal) and 1/4 mapping radius (vertical)
- Dragons and Ghasts are now larger on map radar
- Bugfix that prevented Dragons from showing on the map
- Bugfix (LNP-8PW-SBZV) for in-game map when VoxelMap is loaded
- Bugfix (EQ9-WMJ-4EXQ) for errors when switching to German language
- Updated German translation, thanks to Maik Ploigt!
- Updated Russian translation, thanks to Dzmitry Zhemchuhou!
- Cleaned up English message bundle names, updated Web UI to use new names
- Can now refresh web UI after language changed (without the need to restart Minecraft)
- Added "donate" link to Web UI in the JourneyMap menu
- journeyMap.properties file now uses underscores in properties name. Older files will be converted to use new names.
- Added mcmod.info file for Forge users

Changes in JourneyMap 2.0.1 for Minecraft 1.5:
- Bugfix (XVD-5GX-H9DG) for webserver errors (ZipFileException) on Linux, Mac.  Unzipping the mod no longer needed!
- Added property (update_check.enabled) to disable the check for an updated version to JourneyMap
- Bugfixes in web UI triggered when a newer version of JourneyMap is available
- Bugfix in web UI for "Show Caves" not initially checked
- HTML tweaks in web UI for HTML validation
- Made zoom slider easier to see in web UI
- Optimized CSS in web UI, removed unused selectors and images

Changes in JourneyMap 2.0 for Minecraft 1.5:

- IOS touch event support added to web UI
- Responsive layout tweaks for phone-sized devices
- Day/Night buttons consolidated in Web UI
- Improved readability of world info, location, etc.
- Caves button moved to Show menu
- Removed code for handling obsolete chunk image files
- Bugfix for surface image cache wrongly using altitude in cache key
- Bugfix (78B-SYZ-S2SQ) for cave map depth rendering issue

Changes in JourneyMap 2.0b4 for Minecraft 1.5:

- Filename error inside zip fixed
- Ice color fixed
- Other players now displayed in multiplayer
- Some entity icons resized to be more proportional to the others
- JourneyMap has a new logo
- Added logo splash to web UI
- Removed some obsolete images from zip
- Bugfix to improve file system path determination for web server

Changes in JourneyMap 2.0b3 for Minecraft 1.5:

- New web UI appearance with added support for IE 9
- Faster chunk render cycles for improved game performance
- Faster web server functionality
- Faster web browser scripts
- Improved surface map accuracy and appearance
- Improved cave mapping accuracy and appearance
- Added the ability to toggle display of animals, villagers, pets, other players
- Mobs now show current heading on the map
- Removed blurring (imageSmoothing) when zooming in web UI (except in IE 9, Safari 5, Safari IOS)
- In-game UI now has options screen
- Replaced the embedded servlet engine with a modified version of the NIO webserver "Rupy" (http://code.google.com/p/rupy/)
- Internal work to allow for custom block mods
- The cave lighting property has been re-enabled
- Widened default mapping radius to 5 chunks (was 4)

Known issues:

- Translation updates needed for Russian
- Bug (Tracking ID AW2-6EE-Z3QB): Maps overwrite on multi-world servers because the world seed is no longer provided by the MC server
